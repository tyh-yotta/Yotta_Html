# Yottacontrol 產品資料庫與頁籤維護手冊

所有的產品資訊都集中管理在 `js/data.js` 檔案中的 `productData` 陣列裡。目前產品頁面的內容與右側抽屜頁籤皆採**「資料驅動 (Data-Driven)」**設計，您不需要去修改 HTML 檔案，所有的顯示內容都會根據這份資料庫自動產生。

---

## 📦 單一產品資料結構概覽

一個完整的產品資料會用一組大括號 `{}` 包起來，所有的參數（屬性）之間必須用逗號 `,` 隔開。

```javascript
{ 
    model: "A-1012", 
    image: "assets/A-1012.png",
    desc: "高可靠度的工業控制 I/O 解決方案...",
    desc_en: "High-reliability industrial control I/O solution...",
    // ... 規格與特色 ...
    tabs: [
        // 專屬頁籤設定
    ]
}
```

---

## 📝 參數對照表 (Parameters List)

### 1. 基礎資訊與描述
| 參數名稱 | 說明 | 必填 | 範例格式 |
| :--- | :--- | :---: | :--- |
| `model` | **產品型號**。會顯示在麵包屑與標題上，也作為網址搜尋的依據（如 `?model=A-1012`）。 | ✅ | `"A-1012"` |
| `image` | **產品主圖路徑**。顯示於左側的大圖。建議填寫相對於根目錄的路徑。 | ✅ | `"assets/A-1012.png"` |
| `desc` | **產品簡介 (中文)**。顯示於型號下方的簡短敘述。 | ❌ | `"適用於多種自動化場景。"` |
| `desc_en` | **產品簡介 (英文)**。英文版模式下的簡短敘述。 | ❌ | `"Suitable for various automation scenarios."` |

### 2. 規格表參數 (Specifications)
這部分會對應到產品頁面中的「規格總覽表格」。如果該產品沒有某項規格，可以填入 `"-"` 或留白。

| 參數名稱 | 說明 | 必填 | 範例格式 |
| :--- | :--- | :---: | :--- |
| `di` | 數位輸入 (Digital Input) 通道數。 | ❌ | `"8 ch"` 或 `"-"` |
| `do` | 數位輸出 (Digital Output) 通道數與類型。 | ❌ | `"4 ch (Relay)"` |
| `ai` | 類比輸入 (Analog Input) 通道數。 | ❌ | `"4 ch"` |
| `ao` | 類比輸出 (Analog Output) 通道數。 | ❌ | `"2 ch"` |
| `comm` | 通訊介面種類。 | ❌ | `"RS-485"` |

### 3. 產品特色 (Features)
以「陣列 (Array)」的形式填寫，畫面上會自動把每一項加上打勾圖示。

| 參數名稱 | 說明 | 必填 | 範例格式 |
| :--- | :--- | :---: | :--- |
| `features` | **產品特色 (中文)**。每項特色用引號包起來，逗號隔開。 | ✅ | `["支援 Modbus", "寬電壓輸入"]` |
| `features_en` | **產品特色 (英文)**。對應中文的英文翻譯。 | ❌ | `["Supports Modbus", "Wide voltage"]` |

### 4. 應用領域 (Applications)
顯示於產品「產品應用」頁籤中的標籤按鈕。點擊後會在下方展開對應的解決方案網頁。

| 參數名稱 | 說明 | 必填 | 範例格式 |
| :--- | :--- | :---: | :--- |
| `applications` | 陣列格式。包含 `name` (中文), `name_en` (英文), `link` (網頁路徑)。 | ❌ | `[{ name: "廠務自動化", name_en: "FA", link: "solutions/fa.html" }]` |

### 5. 其他相容參數
供預設頁籤使用的快速下載連結：

| 參數名稱 | 說明 | 必填 | 範例格式 |
| :--- | :--- | :---: | :--- |
| `manualLink` | 使用手冊的直接下載路徑 (PDF)。 | ❌ | `"assets/pdf/manual.pdf"` |
| `specLink` | 規格書的直接下載路徑 (PDF)。 | ❌ | `"assets/pdf/spec.pdf"` |
| `wiringImage` | 接線圖圖片路徑。 | ❌ | `"assets/wiring.png"` |

## 🛠️ 動態抽屜頁籤 (Dynamic Tabs) 進階修改指南

您可以為每一個產品自訂專屬的「側邊抽屜頁籤」與對應的內容，只需修改 `tabs` 陣列。

### 如何修改或新增頁籤？
在該產品的資料中，加入或修改 `tabs` 這個陣列屬性。以下是標準的格式範例：

```javascript
{ 
    model: "A-1012", 
    image: "assets/A-1012.png",
    
    // 👇 就是這個 tabs 陣列
    tabs: [
        {
            id: "tab-docs",
            title: "技術文件",
            content: "<h3>文件下載區</h3><p>這裡可以放下載按鈕的 HTML。</p>"
        },
        {
            id: "tab-wiring",
            title: "接線圖",
            content: "<img src='assets/A-1012.png' alt='接線圖'>"
        }
    ]
}
```

### 頁籤欄位詳細說明
在 `tabs` 陣列中，每一個大括號 `{}` 就代表一個獨立的頁籤。它包含三個必須填寫的屬性：

1. **`id` (頁籤識別碼)**
   - 用來在程式內部對應按鈕與抽屜內容。必須是英文字母或數字，且在同一個產品中**不能重複**（如 `"tab-docs"`）。
2. **`title` (頁籤顯示名稱)**
   - 顯示在右側直式立體標籤上的文字（如 `"技術文件"`）。文字不宜過長。若是雙語版本可加上 `title_en`。
3. **`content` (抽屜內的內容)**
   - 當使用者點擊該頁籤，抽屜滑出時所顯示的完整內容。
   - 支援**完整的 HTML 語法**。若是雙語版本可加上 `content_en`。

> **小技巧：使用反引號 (\`) 換行**
> 如果 HTML 內容很長，強烈建議在 JavaScript 中使用反引號 `` ` `` 來包覆字串，這樣您就可以直接換行，不需要把所有 HTML 擠在同一行。

範例：
```javascript
content: `
    <h3 style="color: #1e6eb8; margin-bottom: 15px;">產品配線圖</h3>
    <div class="wiring-container">
        <img src="assets/A-1012-wiring.png" style="max-width: 100%;">
    </div>
`
```

## ⚠️ 撰寫與修改時的注意事項

1. **標點符號**：請務必確保所有的引號 `""`、逗號 `,` 都是**半形英文符號**。
2. **最後一項不加逗號**：在物件或陣列的最後一個元素後面，建議不要加上逗號以保證相容性。
3. **引號衝突**：如果您用雙引號 `" "` 來包覆 content 字串，裡面的 HTML 屬性 (例如 `class="btn"`) 就必須改用單引號 `' '`，否則程式會報錯。這也是為什麼**建議使用反引號 (\`)** 來包覆外層字串的原因。
4. **預設頁籤機制**：如果您在 `data.js` 裡的某個產品**沒有設定 `tabs` 屬性**，系統會自動幫它套用一套「預設頁籤」（包含技術文件、常見問題與產品應用），因此畫面不會空掉。
5. **樣式排版**：抽屜所有的內容都包覆在 `.inline-drawer-pane` 這個區塊內。如果需要特殊排版，可以直接加上 `style="..."` 行內樣式，或是統一將自訂 CSS 寫在 `product.css` 裡。
