# Yotta 網站維護工具使用手冊

本手冊介紹位於 `MaintenanceTools` 資料夾內的自動化腳本用途與使用方式。這些工具可以協助您防呆、保護共用檔案，以及在網站發布前進行健康檢查。

## 執行方式
在 Windows 環境下，只需要對著 `.ps1` 檔案 **點擊滑鼠右鍵** $\rightarrow$ 選擇 **「使用 PowerShell 執行 (Run with PowerShell)」** 即可。

---

## 1. Check-Links.ps1 (網站死連結自動檢查器)

**用途**：
自動比對產品資料庫 (`data.js`) 中所有的圖片與網頁路徑，確保沒有檔案遺失，避免客戶在網站上看到破圖或 404 錯誤。

**何時使用**：
- 新增或修改了產品模組之後。
- 準備將網站發布/上傳至正式網頁伺服器之前。

**運作邏輯**：
程式會去讀取 `Html_Data/js/data.js`，找出所有 `link`、`image`、`wiringImage` 所指派的路徑，並檢查這些檔案在您的電腦上是否真的存在。如果有遺失，它會顯示紅色的錯誤訊息。

---

## 2. Lock-Shared.ps1 (共用網頁上鎖機制)

**用途**：
保護 `Html_Data/solutions/shared/` 目錄，防止該資料夾內的共用檔案被不小心誤刪。

**何時使用**：
- 平時日常維護時，建議保持在鎖定狀態。

**運作邏輯**：
呼叫 Windows 系統底層的 `icacls` 權限指令，強制撤銷任何人對該資料夾的「刪除權限」。

---

## 3. Unlock-Shared.ps1 (共用網頁解鎖機制)

**用途**：
解除上方的鎖定狀態，讓您可以自由地新增、修改或刪除共用的解決方案網頁。

**何時使用**：
- 當您需要大規模整理、刪除或更新 `shared` 資料夾內的檔案時。
- **注意**：作業完畢後，請記得再次執行 `Lock-Shared.ps1` 將資料夾鎖回安全狀態。
